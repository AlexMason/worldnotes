<objective>
Migrate all 7 existing built-in plugins from the old `Plugin` interface to the new `ContentPlugin` interface. This is an in-place rewrite — each plugin gains `kind: 'content'` and `version: '1.0.0'` fields. No logic, rendering, or token patterns change. The `defaultPlugins` array and test files update their type references.

Purpose: This is a pure type migration. The existing plugins already satisfy `ContentPlugin` structurally — they just need the new required fields. By migrating plugins BEFORE wiring the registry (Plan 03), we isolate type changes from integration changes.

Output: All 7 plugins are valid `ContentPlugin` objects, `defaultPlugins` is `ContentPlugin[]`, and existing tests compile with updated type imports.
</objective>

<!-- ContentPlugin type from Plan 01 — what migrated plugins must satisfy -->
From src/types.ts (updated by Plan 01):
```typescript
interface PluginLifecycle {
  onInit?(): void
  onDestroy?(): void
}

interface ContentPlugin extends PluginLifecycle {
  name: string
  version: string
  kind: 'content'
  tokens: TokenDef[]
  render(token: Token, context: EditorContext): HTMLElement | Text
  onNavigate?(token: Token, context: EditorContext): boolean | void
  onUpdate?(): void
}
```
